# Reproducibility Beyond the Code – organizing research projects in R

## What this workshop covers

Reproducibility is now a baseline expectation in scientific research, yet many projects fall short because they lack clear structure, documentation, and organization. This workshop covers three broad themes:

1.  **Organizing repositories in R** — logical directory structure, separating raw data / processed data / scripts / outputs, and writing scripts that are readable and rerunnable.
2.  **Documentation** — effective READMEs and (likely) lightweight data dictionaries.
3.  **Git & GitHub** — version control basics, recovering earlier versions, safely refactoring or deleting code, `.gitignore`, and GitHub as a tool for meeting journal data/code-availability requirements.

No prior Git or R experience is required, but basic familiarity with research and data analysis is assumed.

## Repository structure

```         
Data/       # raw source data (gitignored) + the committed, anonymized registrants_public.csv
Derived/    # outputs generated by scripts (recreatable, gitignored, kept empty)
Figures/    # saved plots (gitignored except the committed registrants_map.png)
Scripts/    # code
  qmd/      # Quarto slides
```

## Dependencies

To reproduce `01_registrant_map.R` and `qmd/02_slides.qmd`: R (4.4+) with the packages `here`, `dplyr`, `tibble`, `ggplot2`, and `maps`; and [Quarto](https://quarto.org) (1.6+).

The beamer version of the slides additionally needs a LaTeX installation -- [TinyTeX](https://yihui.org/tinytex/) is the easiest route (`quarto install tinytex`), or any existing TeX Live/MacTeX distribution works too.

## Usage

To reproduce the choropleth map and slide deck, the scripts should be run in order from 00 - 02. The 00 wrangling script reads the public registrant's data and formats for use in the 01_plotting script.

- `00b_wrangling.R` - Creates the anonymized, shareable location dataset (first name + last initial, city/state/country) from the raw registrant workbook
- `01_registrant_map.R` — Reads that dataset and plots a US state choropleth of where registrants are from
- `qmd/02_slides.qmd` - Slide deck written in Quarto

## Column names and descriptions

In `Data/registrants_public.csv` there are 5 columns:

- `first_name:` Participants' first name

- `last_initial:` First initial of participants' last name

- `city:` City where participant resides

- `state:` State where participant resides

- `country:` Country where participant resides
