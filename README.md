# Reproducibility Beyond the Code

Materials for a 4-hour workshop at the AOS (American Ornithological Society)
2026 conference: **"Reproducibility beyond the code – organizing research
projects in R."**

Taught by Aaron Skinner, PhD candidate, University of British Columbia.

## What this workshop covers

Reproducibility is now a baseline expectation in scientific research, yet
many projects fall short not because of complex analyses, but because they
lack clear structure, documentation, and organization. This workshop covers:

1. **Organizing R projects & repositories** — logical directory structure,
   separating raw data / processed data / scripts / outputs, and writing
   scripts that are readable and rerunnable.
2. **Documentation** — effective READMEs and lightweight data dictionaries.
3. **Git & GitHub** — version control basics, recovering earlier versions,
   safely refactoring or deleting code, `.gitignore`, and GitHub as a tool
   for meeting journal data/code-availability requirements.

No prior Git or R experience is required — just basic familiarity with
research and data analysis. This repository is itself organized the way the
workshop teaches, so it doubles as a live example.

## Repository structure

```
Data/       # raw, unmodified source data
Derived/    # outputs generated by scripts (recreatable, gitignored)
Figures/    # saved plots (gitignored unless a static/final asset)
Scripts/    # analysis code
  qmd/      # Quarto manuscript/slide files
```

## Slides

Slides are written in [Quarto](https://quarto.org) (`Scripts/qmd/slides.qmd`)
and render to reveal.js HTML:

```bash
quarto render Scripts/qmd/slides.qmd
```
